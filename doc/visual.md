# Dungeon Shift / 地牢换班 — Visual Bible

## 1. Visual thesis

- 游戏与受众：面向移动端轻中度玩家的异步造关/破关 3D 潜入游戏。
- 情绪承诺：像俯视一座真实玩家亲手搭建的危险桌面地牢，读懂其节奏后完成一次干净盗窃。
- 一句话视觉论点：**一台架设在午夜地牢模型旁的像素作战终端：3D 世界清晰、克制，界面像被地牢主反复敲打过的专用控制器。**
- 标志性瞬间：潜入者夺取琥珀宝物，整个地牢短暂熄暗，青金冲击环穿过地格，所有怪物同时转头。
- 三项必须品质：地格与视线一眼可读；角色轮廓在 160 px 宽画面仍可辨认；UI 与石材/金属世界属于同一套设计。
- 三项禁止方向：写实恐怖与血腥；紫蓝霓虹赛博朋克；通用玻璃卡片覆盖 3D 场景。

## 2. Composition and camera

- 方向与比例：竖屏 9:16 优先；3D Canvas 填满视口，DOM HUD 使用安全区适配。
- 相机：正交 3/4 等距相机，世界方向约 `(6.4, 10.8, 10.2)`，俯角约 43°；轻微偏前以让 5×7 长地牢更接近竖直构图，并使入口与宝库同时留在手机视野内；不允许游戏中自由旋转。
- 入场镜头：每局用 1.6 秒完成一次约 22° 的克制水平环绕，从 0.86 倍全局视距与宝库侧焦点开始，扫过用户设计的墙、机关和守卫后落到入口玩家；镜头结束前不启动计时或输入。它是关卡展示，不是可操控的自由旋转。
- 玩法焦点：5×7 地牢占画面中央约 68% 高度；入口位于下三分之一，宝库位于上三分之一。
- 前景：潜入者、脚下目标格、短时粒子；中景：墙、机关、守卫和视线；背景：近黑展台、薄雾与远处低亮石柱剪影。
- HUD 安全区：顶部避开 `env(safe-area-inset-top)` 后 12 px；底部控件避开 `env(safe-area-inset-bottom)` 后 12 px。
- 注意路径：宝库琥珀光 → 潜入者青色轮廓 → 守卫暗红视线 → 顶部时间与警戒。

## 3. Color — Pixel Operations Palette

- 总体配比：靛黑/土褐中性色 75%，行动青 10%，收益琥珀 8%，警戒洋红 4%，危险红、成功绿与符文紫合计 3%。高饱和色一次只允许一个成为视觉主角。
- UI 中性色阶：虚空 `#05070B`、画布 `#080C12`、基础面 `#0D121A`、抬升面 `#131923`、弱边框 `#202735`、强边框 `#353D4A`、弱文字 `#9EA39F`、正文 `#E8E2D2`、强文字 `#F7F0DC`。
- 结构土褐：最深 `#180D08`、面板侧面 `#28160D`、内框 `#3B2415`、受光边 `#56351D`。它只承担地牢材质感，不代表按钮状态。
- 行动青阶：`#082B36 / #0B4355 / #0E6078 / #147F9D / #1FA5C9 / #4CC7DE / #83E2EE`。主操作使用 `#4CC7DE` 配 `#041015`；焦点框与已装备状态用 `#83E2EE`。
- 收益琥珀阶：`#3B2107 / #5C3409 / #85500E / #B8781A / #D99B35 / #F1C66B / #FFE0A0`。只用于宝物、分数、预算与成功结算，不用于普通导航。
- 警戒洋红阶：`#39101F / #5A1730 / #812044 / #AF315E / #CF4B76 / #E77D9B`。只用于警戒槽和锁定提示，避免与伤害红混用。
- 伤害红阶：`#3B100C / #7F241A / #AA3426 / #CF4A38 / #EB7464`；成功绿阶：`#14280F / #2F5C22 / #488537 / #67A94E / #8BC875`。
- 符文紫 `#8E4DB3` 及暗面 `#32143F` 只用于世界中的魔法符文，不进入常规按钮体系。
- 环境灯仍使用靛蓝 `#6F7CFF`、入口冷青、宝库琥珀与守卫酒红；它们负责照亮 3D 空间，不能被当作 DOM 控件底色。危险红与宝物琥珀不得照在同一主体上。
- 不使用连续渐变、玻璃透明渐变和通用外发光。只允许用 1–2 px 分层色带、棋盘抖色或离散段格表现像素材质与进度。

## 4. Typography

- 像素标签、英文按钮和关键数字：本地打包 `Press Start 2P`，400。它只用于 9–15 px 的短文案，中文按钮自动回退清晰的系统黑体，避免把长正文变成难读的像素块。
- 中文动作文字不使用小号像素字：主按钮固定 13 px/600，次按钮与标签页 12 px/600，技能与紧凑操作 11 px/600–700，五列建造工具最低 10 px/600。亮青按钮统一使用深靛字 `#041015`，禁止高明度底色叠加白字。
- 英文展示标题与大数字：本地打包 `VT323`，400；中文展示回退 `PingFang SC / Noto Sans SC`，使用 600 而不是粗黑字重。
- 正文：打包 `IBM Plex Sans` 400/500/600，中文回退 `PingFang SC / Noto Sans SC`。
- 字阶：微标签 8/12 px、工具标签 9/12 px、正文小字 12/18 px、正文 15/22 px、卡片标题 20/22 px、屏幕标题 30/30 px、结算标题 42/40 px、开始标题 58/52 px。
- 像素字体不加描边；长中文正文不加宽字距。Press Start 标签允许 `0.04em`，大写短标签允许 `0.08em`。数字统一等宽排布。

## 5. Shape, material, and lighting

- 主形状：正方地格、45° 斜切楔形、低段数圆柱、粗短机械卡扣；UI 统一 0 px 圆角，外轮廓保持阶梯式直角。
- 边框与阴影：2 px 外框 + 1 px 内高光；主面板 4 px 硬阴影，主按钮 5 px 硬阴影；按下向下位移 3 px 并收短阴影，不使用柔光。
- 材质：人物与道具使用 flat-shaded `MeshStandardMaterial`；石材粗糙度 0.82；青铜粗糙度 0.58、金属度 0.28。
- 光线：右前上方暖桃色主光负责形体，左后方靛蓝方向光统一夜色；入口提供低位冷青点光与灯池，宝库提供高位琥珀点光与灯池，守卫携带短距离酒红移动光。三种局部灯光互不重叠成为一片彩雾，三值面仍保持顶面 1.0、近侧 0.80、远侧 0.62。
- 灯光强度层级：暖桃主光 2.6；靛蓝方向填充 0.70；入口青光 4.2/距离 4.8；宝库琥珀光 4.0/距离 5.2；守卫红光待机 0.8、追捕 2.4/距离 2.5。
- AO 与接触阴影必须存在，角色脚底不可漂浮。

## 6. Characters, environments, and assets

- 潜入者优先复用 `student`、`teen`、`worker`、`punk`、`cowboy`；守卫复用 `vampire`、`werewolf`、`zombie`、`ghost`、`skeleton`、`mummy`。
- 人类比例维持现有低多边形家族；怪物靠轮廓和前突识别特征区分，不做贴图换色替代。
- 四肢通过已有 `userData.rig` 暴露的肩/髋枢轴动画；新增待机、行走、警觉、追捕、受击和夺宝姿态。
- 地牢模块遵循 0.125 世界单位网格，只使用硬边盒、低段圆柱、多面球、锥和楔形。
- 可读细节不得小于 0.125 世界单位；机关英雄特征至少 0.25 世界单位。
- 资产使用 GLB；UI 预览使用透明 PNG sprite；图片不可拖动且不设跨域属性。

## 7. UI, icons, and screen layout

- 图标为同一套 24×24 SVG 硬角线性图标，2 px stroke、方形端帽：烟雾、冲刺、返回、重试、声音、排行榜、地牢主、潜入者。
- 主按钮高 56 px，次按钮高 50 px；图标按钮至少 48×48 px；所有触控目标至少 44×44 px。
- 开始页：标题位于上部安全区，任务档案固定在下部；“潜入”独占一行，建造与档案二等分，排行榜作为标题栏独立像素图标键。主次关系靠面积和色彩共同表达。
- 游戏 HUD：顶部为紧凑三栏作战条，警戒槽拆成离散像素段；目标提示悬于其下。底部技能键使用同一宽度和基线，不遮住入口地格。
- 建造页：56 px 页头、居中的 5×7 蓝图、五等分工具带、单行校验反馈、底部发布。蓝图在 390×844 最大宽 288 px，在 360×640 收缩到 224 px，不能挤压工具触控区。
- 档案页：页头、说明、双标签、滚动卡片与底部建造键分层；卡片小地图固定约 96×134 px，内容列使用 `minmax(0,1fr)`，长用户名不改变列宽。
- 结算页：底部像素舞台面板，顶端悬挂胜负徽记；分数最大、三项指标其次，重试为唯一青色主按钮。
- 面板使用深靛正面 + 土褐结构边；选中状态用行动青内框与底边，不使用泛光。默认、焦点、警戒、成功分别使用青、青、洋红、琥珀/绿色，颜色之外还必须有边框或文案差异。
- 加载以三格依次点亮；警戒以洋红色双短脉冲；成功用琥珀色离散色块从左到右扫过。所有动效采用 `steps()` 或 2–4 帧定格节奏。
- Emoji 绝不充当功能图标。

## 8. Motion and VFX

- 动作性格：短促、机械、略带桌面模型的定格感；常规过渡 160–240 ms，重要结果 420–750 ms。
- 行走：左右腿/手臂反相摆动，单步周期 440 ms，身体上下 0.04 世界单位；落脚帧有尘粒。
- 警觉：守卫在 110 ms 内抬头，停顿 55 ms，然后转向；追捕步频提高 35%。
- 受击：55 ms 冻结、90 ms 闪白、140 ms 最大 3 px 震动、420 ms 恢复；减少动态模式取消冻结与震动，仅保留色块和粒子数量减半。
- 粒子均为硬边体素或低多边形环；普通 3–5 个，成功 24 个，寿命不超过 750 ms；不使用圆形软光点雨。
- 持续视线使用透明平面锥但不叠加 Bloom；只有锁定边缘在 2 Hz 下脉冲。
- 音画冲击峰值误差目标小于 35 ms。

## 9. References translated into principles

- 现有 `_lowpoly_lab`：复用正交等距、硬边平面着色、三值光照、单一青色家族强调和缩略图可读原则；不照搬便利店题材。
- 现有 `_shared/engine-3d`：复用角色肩髋枢轴、体素粒子池和固定渲染基础；不照搬跑酷关卡或办公室物件。
- 桌面立体模型：使用明确底座、接触影和有限景深层次；不加入写实木纹、塑料反光或微缩摄影噪点。
- 经典潜入游戏：使用视线、警戒节奏和安全窗口；不复制其 UI、角色或关卡布局。

## 10. Anti-patterns

- 禁止混入平滑高模、写实 PBR 人物、卡通描边 2D 角色、Emoji 图标或照片头像作为 3D 角色。
- 禁止大面积 Bloom、紫蓝渐变天空、通用玻璃拟态、每个元素都有圆角卡片和持续屏幕抖动。
- 禁止把危险只表达为颜色；视线锥、惊叹形轮廓与音效必须共同传达。
- 禁止相机自动旋转、游戏中双指缩放或让 HUD 随 3D 场景倾斜。
- 禁止过小机关、薄如纸的墙、没有接触影的角色和超出地格体系的任意摆放。

## 11. Vertical-slice acceptance

- 入口：3 秒内看懂“潜入样板地牢”，能看到真实 3D 人类潜入者与怪物守卫，不靠标题解释题材。
- 游戏：5×7 地牢、宝物、出口、一个巡逻守卫、一个尖刺和一个警报符文在 390×844 上同时可读。
- 高反馈：触发守卫锁定时必须出现姿态变化、红色视线脉冲、短冻结、体素碎片和同步警报音。
- 完成：夺宝并回到出口后显示结果、时间、生命和再次挑战；失败能指出具体原因。
- 窄屏：360×640 无横向溢出，技能按钮不遮挡入口地格，正文不小于 16 px。
- 性能：中档手机目标 50–60 FPS；像素比上限 1.75；粒子池上限 96；局部点光固定为入口、宝库和守卫 3 个，只有守卫灯移动。
- 视觉 QA：首次截图后修复全部 P0/P1，并以相同状态和视口复验后才扩展建造与社交系统。

## 12. Gate D — 建造与社交界面

- 建造页把 5×7 地牢转译为同材质的平面蓝图：暖灰地格、冷青入口、琥珀宝库、砖红守卫/符文与白色尖刺，保持与 3D 场景相同的语义颜色。
- 顶部预算以大号等宽数字和“点剩余 / PTS LEFT”双通道表达；底部五件工具使用同一套 24×24 硬角 SVG，选中态同时使用冷青边框、底边和文字，不只依赖颜色。
- 发布前状态包含：未布置守卫、路径被封、预算不足、入口/宝库不可修改、路线验证成功与按钮禁用；发布成功在档案页用左侧冷青竖线状态条确认。
- 挑战档案采用纵向可滚动卡片；每张卡同时展示小型关卡图、预算、守卫数、名称、作者头像/名字与明确的“潜入 / RAID”操作。自己的内容显示“你 / YOU”，不伪造头像。
- 社区档案卡片中的作者头像与名字可进入 AlterU 资料；结算页再次显示同一作者身份。普通浏览器显示下载 AlterU 的明确降级入口。
- 排行榜沿用同一石板、旧金与冷青体系；不使用奖牌 Emoji。加载、空、平台外降级与成绩列表均有独立状态。
- Gate D 验证视口为 390×844 与 360×640；编辑器、发布成功、我的档案、社区降级、排行榜、游戏中、夺宝与结算均需截图，全部交互目标至少 44×44 px。

## 13. Pixel direction boundary

- 像素化只覆盖 DOM UI、地图编辑器和反馈特效的节奏；Three.js 场景、低多边形角色、材质和灯光保持干净渲染，不对 Canvas 做全局 `image-rendering: pixelated`。
- UI 表面允许极低对比度的 4×4 棋盘抖色和 1 px 扫描纹，但不允许覆盖角色面部、机关轮廓或文字。
- 风格不是蓝屏、黑客终端或赛博朋克：不使用满屏亮蓝、不堆叠等宽代码、不增加无功能的故障字符。
- 细腻感来自严格的颜色职责、对齐、层级和内外边结构，而不是增加更多纹理、颜色或装饰。

## 14. Approved review fidelity contract

- `review/ui-pixel.html` 的 REV 04 像素方案是正式界面的结构基准，不只是一张情绪参考图。实现不得自行替换其展示标题、任务卡层级、HUD 分段、地图比例、威胁条或结算顺序。
- 品牌与屏幕展示标题固定沿用设计稿英文像素字：`DUNGEON SHIFT`、`SET THE AMBUSH`、`CHOOSE A KEEPER`、`CLEAN GETAWAY`、`STAGE CLEAR`；中文用于说明、状态、动作按钮和无障碍名称。
- 首页必须保持“两行品牌标题 → 一句行动承诺 → 3D 主体 → 底部 MISSION FILE 卡”的构图；任务卡显示 `ARCHIVE 01`、RANK、守关者、75 秒、一个青色主操作和两个深色次操作。
- HUD 必须显示 RUN 编号、TIME/战利品/HP、8 段警戒槽、目标条和两个带 READY/USED 子状态的技能键。
- 建造地图在 390×844 固定为 270×378 px，带 `FLOOR 07` 标签；档案卡带 5 段威胁条；结算依次为 STAGE CLEAR、遗物、结果标题、档案/作者、分数、三项指标和两个动作。
- 真实内容数量可以与 mockup 不同：社区只有一份地牢时只显示一张卡，不得伪造第二位用户来填满设计稿。

## 16. Gameplay readability contract

- 镜头不是全图固定监控视角：它以克制幅度和 180 ms 平滑时间跟随潜入者，让主角始终成为画面焦点，同时保留前方 2–3 格决策空间。冷青重点光附着于主角，固定入口青光与宝库琥珀光仍保留为地图语义锚点。
- 守卫模型的面部朝向定义为本地 `+Z`，视线锥也从本地 `+Z` 展开。禁止用模型旋转补偿一个沿 `+X` 建立的视线图形；逻辑检测必须读取同一前向向量。
- 视线由三层组成：低透明度地面填充、亮红硬边、暴露时的头顶几何感叹标记。它必须被墙体视觉遮挡，不能像投影贴纸一样穿墙；危险不能只靠透明红色表达。
- 夺宝后入口成为画面第二焦点：入口灯池加强，主角脚下出现指向入口的冷青箭头，HUD 使用“宝物到手 · 跟随青色箭头返回入口”。箭头只在携宝撤离阶段出现，结算后立即消失。
- 通关反馈顺序固定为“进入入口 → 输入锁定 → 青色成功环 → 下一动画帧弹出结算”。禁止在地图上停留 500 ms 以上才出现结算，也禁止只播放世界特效而没有可返回的 DOM 面板。
- 死亡反馈固定为“55 ms 定格 → 镜头在 360 ms 内推进至 1.26 倍并锁定仍完整的角色 → 推进到位后玩家才红色体素爆散并熄灭 → 红黑像素闸门合拢 → 第 860 ms 入口重亮 → 镜头在 520 ms 内拉远至 1.00 倍并平滑迁回主角”。禁止镜头尚未到位就先炸裂。镜头使用三次缓出，不允许在复活瞬间跳焦；随机震动只保留致死接触的前 90 ms。闸门只覆盖 Canvas 层；减少动态模式把推进和拉远分别缩短为 180 ms，但保留先定格、后爆裂的事件顺序。
- 元素识别采用固定的双通道词汇，不靠单一色相：潜入者是“青色脚环 + 头顶菱形”，入口是“四角门柱 + 青色灯池”，宝物是“琥珀晶体 + 竖直信标”，守卫是“红色脚环 + 朝脸展开的硬边视线”，尖刺是“深红底板 + 高锥体”，符文是“紫色方环 + 悬浮晶体”。这些特征在 360×640 截图中也必须保留可辨轮廓。
- 决策层级固定为：当前角色与可走格 > 即时危险视线 > 当前任务目标（宝物或出口） > 静态机关 > 墙与装饰。相邻可走格只显示细青色硬边方框；任何时候不得让所有地格同时高亮。
- 颜色语义保持唯一：行动青只用于玩家、可走格和出口；琥珀只用于战利品；红色只用于守卫与直接伤害；紫色只用于警报符文。每个语义必须再配合形状、位置或动态变化，确保灰度下仍可区分。
